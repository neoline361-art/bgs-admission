# BGS Admission Connect

A complete, production-ready admission management web application for BGS PU College, Hanumanthpura, Shidlaghatta, Karnataka 562105.

## Run & Operate

- `pnpm --filter @workspace/bgs-admission run dev` — run the frontend (port auto-assigned)
- `pnpm --filter @workspace/api-server run dev` — run the API server (port 8080)
- `pnpm run typecheck` — full typecheck across all packages
- `pnpm run build` — typecheck + build all packages

## Stack

- pnpm workspaces, Node.js 24, TypeScript 5.9
- Frontend: React 19 + Vite + Tailwind CSS (Wouter routing)
- Backend: Supabase (PostgreSQL) with Row Level Security
- Export: SheetJS (CSV, Excel, ODS)
- QR Code: qrcode.react

## Architecture decisions

- Supabase anon key used for all operations (appropriate for internal school tool)
- Staff login uses SHA-256 PIN hashing (Web Crypto API, no external libraries)
- Sessions stored in sessionStorage (clears on browser close)
- Real-time updates via Supabase Postgres Changes
- CAPTCHA is visual digit distortion (no math, no external service)
- Application IDs auto-generated by Postgres trigger: BGS-2026-0001, etc.

## Product

Students scan a QR code → apply without login → get an Application ID.
Staff login with username + PIN + CAPTCHA → access role-based dashboards:
- **Teacher**: View assigned applications, update status, WhatsApp, add notes
- **Office**: All applications, bulk actions, assign teachers, fix meetings, export
- **Admin**: Analytics (village/course/teacher summaries), staff management, database backup

## User preferences

- Zero premium tools — everything free forever
- Mobile-first design with large text and big buttons
- Blue primary color: #1e40af
- No animations, no gradients — clean and simple
- English default (Kannada toggle can be added later)

## Gotchas

- Must set VITE_SUPABASE_URL and VITE_SUPABASE_ANON_KEY before the app is functional
- Run supabase/schema.sql in Supabase SQL Editor before first use
- Staff PINs are hashed with SHA-256 + salt "bgs_salt_2026" — use Admin Dashboard to add staff
- LockManager warning from Supabase is non-critical (multi-tab sync) — ignore it
